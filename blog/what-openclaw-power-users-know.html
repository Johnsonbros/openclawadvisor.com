What OpenClaw Power Users Know That You Don't

February 12, 2026 • 9 min read

There's a gap between the people hyping OpenClaw on Twitter and the people actually running it to make money.

I spent 27 hours digging through Reddit threads, Hacker News comments, GitHub issues, security reports, and YouTube teardowns. Not the sanitized blog posts. The real conversations where people share what's actually working.

Here's what I found: The people getting real value treat OpenClaw like infrastructure—isolated, hardened, and purpose-built for specific workflows. The people getting burned treat it like a consumer app.

Let me show you what the power users are doing differently.

1. They Don't Use Built-In Memory (At Least Not Alone)

OpenClaw ships with a RAG system using sqlite-vec for vector search. It's hybrid—semantic search plus BM25 keyword matching, merged with Reciprocal Rank Fusion. Sounds good on paper.

But here's the catch: memory compaction only works well on Anthropic models. If you're using Gemini, you hit context bloat fast because pruning isn't implemented for non-Anthropic providers. One power user described their context growing to 200k tokens in a single session. At that point, the agent becomes useless.

What Power Users Do Instead

They run external memory systems:

  • ClawRag (production-ready): Self-hosted RAG engine with ChromaDB vector storage, 512-token chunks with 128-token overlap, Docker-first deployment. Handles hundreds of documents with semantic search.
  • Supermemory (cloud-hosted): Requires a Pro subscription but auto-recalls relevant memories before every AI turn, auto-captures conversations after each turn, and builds persistent user profiles.
  • Manual curation: They edit MEMORY.md files aggressively to prune noise. This is boring but effective.

The pattern that works: short-term memory (OpenClaw's built-in RAG) for the current session, long-term memory (external system) for historical context, and manual curation to keep things clean.

✓ One HN user said it best:

"Memory works surprisingly well. Daily markdown logs + semantic search. It references decisions from days ago and builds on previous work. The magic isn't any single feature—it's having a persistent assistant that remembers context across sessions."

2. They Run on VPS + Tailscale, Not Their Laptop

Most OpenClaw tutorials show you how to install it on your MacBook. That's fine for tinkering. But power users running real businesses? They're not doing that.

The VPS Pattern (from multiple Reddit and HN threads):

  • Run OpenClaw in Docker on a small VPS ($5-10/month)
  • Tailscale for secure access (not exposed to the internet)
  • Separate Gmail account for email integration (burner strategy)
  • Delegate access instead of full OAuth tokens
  • Auto-label sensitive emails (banking, crypto) and configure agent to skip them

Why VPS instead of laptop? Because your laptop isn't always on. Power users need 24/7 operation. They wake up to work their agent already did while they were asleep.

One guy described his morning briefing setup: "I wake up to a full briefing. My agent already did 4 hours of work while I was drooling on my pillow." That's the standard.

3. They Use the "Supervisor Pattern"

Beginners run one agent and try to make it do everything. Power users run multiple specialized agents orchestrated by a supervisor.

Real example from HN: A developer runs an agent called "Patch" that manages 5-10 specialist agents via shared tmux. He can SSH into his workstation from his iPhone while standing in line at Disneyland and manage 20 open terminals through Patch.

Another user runs Buddy (personal assistant) who manages:

  • Katy: Twitter/X growth
  • Jerry: Job scouting
  • Burry: Crypto trading
  • Mike: Security monitoring
  • Elon: Building and shipping features

Agents share intelligence through a shared file, coordinate in a Telegram group, and produce overnight results. The human supervises the supervisor, not the individual agents. This is how you scale.

4. They Write Detailed Specs (The "Bead" System)

One power user on HN runs a 22-core workstation with multiple Claude Code instances. His secret? Spec-driven development with "beads."

The process:

  1. Spec out everything in detail with docs
  2. Create granular tasks called "beads" (dependency structure)
  3. Agents pull a bead to implement
  4. Custom linters and code standardization docs enforce quality
  5. Pre-commit hooks catch garbage code

His assessment: "Better code than I could write after 10 years of focused daily coding."

This is the difference between vibe coders and professionals. Vibe coders say "build me a website" and get a broken React app with hardcoded API keys. Professionals write a 12-page spec, break it into 40 beads, and let agents execute with precision.

5. They Aggressively Limit Tool Permissions

OpenClaw ships with near-unlimited permissions by default. exec can run arbitrary shell commands. write can modify any file. browser can access any website.

This is terrifying in production. One prompt injection attack could delete your database.

What power users do:

  • Principle of least privilege: Only grant tools agents actually need
  • Scope permissions: exec whitelists allowed commands, read/write restricted to specific directories, browser limited to approved domains
  • Sandboxing: Run agents in Docker containers with read-only filesystem mounts and network restrictions
  • Approval workflows: High-risk actions (database writes, external API calls) require explicit human approval

⚠️ Real Story

One power user's agent sent an aggressive email to their insurance company after misinterpreting context. It accidentally triggered a renewed investigation instead of a rejection. They now have a rule: all emails to specific domains require approval.

6. They Control Costs Like Their Business Depends On It

Because it does.

AI consultant Shelly Palmer spent $250 on setup alone. $400/month is common for power users ($200 Claude + $200 OpenAI). Cost optimization becomes a full-time job.

Power user cost strategies:

  • Model switching: Opus for planning and hard problems, Haiku for execution and simple tasks
  • Prompt caching: Save 90% on repeated context (Anthropic-specific)
  • Kill sessions aggressively: When context exceeds 100k tokens, start fresh
  • Budget caps: $20 weekly budget, reload manually to force awareness

If your agent costs $400/month to run, your services need to generate way more than that to justify it. Power users track token costs per task and optimize ruthlessly.

7. They Paper Trade Before Going Live

This applies to trading, but the principle works everywhere: test in simulation before betting real money.

One trader ran their OpenClaw strategy in paper trading for weeks, hitting a 77% win rate before going live. Others went live too quickly and lost money.

For non-trading use cases, this means:

  • Test email automation on a dummy account first
  • Run customer service replies in "draft mode" for human review
  • Deploy to staging before production
  • Monitor closely for the first 30 days

8. They Have a Kill Switch

Power users running agents 24/7 have a way to immediately shut everything down.

Common patterns:

  • Telegram command (/pause trading)
  • API endpoint to disable execution
  • Docker container stop script
  • Manual override that requires daily re-enable

One Mac Mini user running a business said: "Too agreeable by default. Had to explicitly program 'push back on bad ideas' rules into its personality file."

Agents will hallucinate revenue figures, invent specs, and implement bad ideas without questioning. The kill switch prevents runaway disasters.

The Money Question: Are People Actually Profiting?

Yes. But not the way you think.

Proven revenue models from Reddit and real case studies:

  1. White-label AI infrastructure for agencies ($2k-5k/month): Marketing agencies outsource technical implementation. You provide OpenClaw setup as white-label service. Recurring revenue for maintenance plus API cost markup.
  2. Custom workflow automation ($5k-10k one-time): Business process automation, API integrations, CRM connections. High margin (80%+), repeatable process.
  3. Content services ($500/month): Blog posts, social media management, SEO content pipelines. Brand voice maintenance.
  4. Freelancer 10x capacity: Not a service—a personal multiplier. Handle 10x normal client load by automating repetitive tasks.

One real example: A guy running a SaaS plus AV rentals business in Orange County. Mac Mini M4 running Opus 4.5 via Claude Max. Overnight work includes research reports, competitor analysis, lead lists. Found and fixed a broken SMS chatbot (broken for 10 months). Connected to 4 new APIs in one session.

Revenue generated: $0 (yet). But work that wouldn't have been done.

That's the real value. Not replacing your job. Doing the work you never had time for.

What This Means for You

If you're trying to use OpenClaw like a ChatGPT wrapper, you're going to be disappointed. The power users aren't doing that.

They're treating it like infrastructure: isolated environments, hardened security, purpose-built workflows, detailed specifications, cost controls, and human oversight.

It's not plug-and-play. It's build-and-maintain. But if you're willing to do the work—or hire someone who knows how—the upside is real.
